Organizing Specs

Organizing Specs

In this lesson I learnt about organizing specs for my unit tests. Organizing specs assists in making our unit tests clear and readable. This will improve maintainability and workflow of the unit tests. There are a couple of things to consider when organizing specs, which are:

  1. Using describe to group our specs(considered as a suite)
  2. Using naming conventions. This involves creating a spec file with the same name as the source file we are unit testing. e.g calculator.js -> calculator.spec.js
  3. Creating the spec file in the same folder as the source file we are testing
  4. Nesting suites

Nesting suites

Nesting suites makes it easier to know which group of specs tests what from the source file. This helps organise our unit tests and makes it readable for other team members to follow. For example in the calculator.spec.js file, we can have a general suite for the Calculator class, another for the add() method of the class and so on.

Setup and Teardown

In jasmine, a setup is where you place prerequisites for your specs and it executes before specs are run. Methods like beforeEach and beforeAll are used for setups. The beforeEach method is executed before each spec in the suite in which it is called. Which means if we have 10 specs, beforeEach will be run 10 times(once for each spec). The beforeAll method is executed once before all the specs in the suite in which it is called.

Teardown is where clean up steps for our specs are defined and it executes after specs are run. Jasmine offers two methods for teardown, afterEach and afterAll. afterEach is executed after all specs in the suite in which it is called. The afterAll method is executed once after all the specs in the suite in which it is called. 

  1. // calculator.spec.js
  2. describe('Calculator', () => {
  3.   let calculator;
  4.   let calculator_two;
  5.  
  6.   beforeEach(() => {
  7.     // we can use this method to define all variables that are common in our specs in the suite
  8.     calculator = new Calculator();
  9.     calculator_two = new Calculator();
  10.   });
  11.  
  12.   afterEach(() => {
  13.     // runs after each spec is run in the suite
  14.   });
  15.  
  16.   it('should initialize the total', () => {
  17.     expect(calculator.total).toBe(0);
  18.   });
  19.  
  20.   // More Specs...
  21. });
  22.  
  23. // main.spec.js
  24. describe('updateResult()', () => {
  25.   let element;
  26.  
  27.   beforeAll(() => {
  28.     // executed once before specs are run
  29.     element = document.createElement('div')
  30.     element.setAttribute('id', 'calculator-result-span')
  31.     document.body.appendChild(element)
  32.   })
  33.  
  34.   afterAll(() => {
  35.     // in this case we use afterAll to clean up the DOM after the spec has run and passed
  36.     const element_remove = document.getElementById('calculator-result-span')
  37.  
  38.     document.body.removeChild(element_remove)
  39.   })
  40.  
  41.   it('adds result to DOM element', () => {
  42.     updateResult('5')
  43.  
  44.     expect(element.innerText).toBe('Result: 5')
  45.   }) 
  46. })
Calculator UI

Above, we added a spec file(main.spec.js) for the script(main.js) that handles the event listeners of the calculator UI.

Try it below:

Result: 3 
  1. // main.js
  2. const calculate = (input_value) => {
  3.     /**
  4.      * Below regural expression(regex) is for matching our operators (+, -, * and /).
  5.      * In JavaScript we wrap regexes with a slash in the beginning and end of the expression.
  6.      * This regex was generated here: https://regex101.com/
  7.     */
  8.     const expression = /\+|\-|\*|\//;
  9.  
  10.     // Here we split the string from the input into an array
  11.     const numbers = input_value.split(expression);
  12.  
  13.     /**
  14.      * Get the first and second number in the arithmetic operation.
  15.      * Because they are strings we use parseInt() to convert them to numbers.
  16.     */
  17.     const first = parseInt(numbers[0]);
  18.     const second = parseInt(numbers[1]);
  19.  
  20.     /**
  21.      * The match helper is a string prototype that retrieves the result of matching a string against a
  22.      * regex. The result of it is returned as an array. In this case it will match the operator and return it.
  23.      * 
  24.      * More info: https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/String/match
  25.     */
  26.     const operator_match = input_value.match(expression);
  27.     let operator;
  28.  
  29.     if(Number.isNaN(first) || Number.isNaN(second) || operator_match === null) {
  30.         return updateResult('Operation not recognized');
  31.     } else {
  32.         // get the operator from the match result array
  33.         operator = operator_match[0];
  34.     }
  35.  
  36.     const calculator = new Calculator();
  37.     calculator.add(first);
  38.  
  39.     let result;
  40.  
  41.     switch (operator) {
  42.         case '+':
  43.             result = calculator.add(second);
  44.             break;
  45.         case '-':
  46.             result = calculator.subtract(second);
  47.             break;
  48.         case '*':
  49.             result = calculator.multiply(second);
  50.             break;
  51.         case '/':
  52.             result = calculator.divide(second);
  53.             break;
  54.         default:
  55.             break;
  56.     }
  57.  
  58.     return updateResult(result);
  59. };
  60.  
  61. const updateResult = (calculator_result) => {
  62.     const calculator_result_element = document.getElementById('calculator-result-span');
  63.  
  64.     if (calculator_result_element) {
  65.         calculator_result_element.innerText = `Result: ${calculator_result}`;
  66.     }
  67. };
  68.  
  69. if (typeof(document.getElementById('calculator-input')) !== 'undefined' && document.getElementById('calculator-input') !== null) {
  70.     document.getElementById('calculator-input').addEventListener('keyup', (event) => {
  71.         if (event.key === 'Enter') {
  72.             return calculate(event.target.value);
  73.         }
  74.     })
  75. }

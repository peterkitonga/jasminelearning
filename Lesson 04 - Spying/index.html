Spying on Your Code

Spying on Your Code

In this section I learnt about working with spies. Spies in jasmine are used to isolate dependencies that a particular method or class that a spec is testing uses. Spies also help create test doubles which are objects that can stand in for real objects in a test. A spy can stub any function and tracks calls to it and all arguments. A spy only exists in the suite(describe) or spec(it) block in which it is defined and will be removed after each spec. There are special matchers used for interacting with spies which include:

  1. toHaveBeenCalled
  2. toHaveBeenCalledWith
  3. toHaveBeenCalledTimes

The 'toHaveBeenCalled' matcher

This matcher tests whether the defined spy has been called. We use this together with a spy that has been defined in the spec or suite. Consider the example below where we test if the installed updateResult spy is invoked. 

  1. it('validates expression when first number is invalid', () => {
  2.   /**
  3.    * The spyOn() method is a global jasmine method that accepts two arguments: the object to spy on
  4.    * and the method name to replace with a spy
  5.    * 
  6.    * Here, we spy on the updateResult method which is available globally in the window object.
  7.    * The '.and.stub()' simply tells the spy to do nothing when invoked
  8.    * 
  9.   */
  10.   spyOn(window, 'updateResult').and.stub()
  11.  
  12.   calculate('a+3')
  13.  
  14.   /**
  15.    * This simply checks that our spy has been called atleast once in the
  16.    * calculate method call
  17.   */
  18.   expect(window.updateResult).toHaveBeenCalled()
  19. })
The 'toHaveBeenCalledWith' matcher

This matcher tests whether the argument passed to the invoked spy matches the expected value. This expected value is passed to the matcher as an argument as well. 

  1. it('validates expression when operation is invalid', () => {
  2.   spyOn(window, 'updateResult') // and.stub() can be omitted because it is the default behaviour
  3.  
  4.   calculate('3_3')
  5.  
  6.   /**
  7.    * checks that our spy has been called atleast once in the calculate method call
  8.   */
  9.   expect(window.updateResult).toHaveBeenCalled()
  10.  
  11.   /**
  12.    * checks if the spy has received an argument of the expected value passed
  13.   */
  14.   expect(window.updateResult).toHaveBeenCalledWith('Operation not recognized')
  15. })
The 'toHaveBeenCalledTimes' matcher

This matcher accepts an argument that specifies the expected number of times our spy is invoked. This simply tests the number of times our spy has beene called. 

  1. it('validates expression when second number is invalid', () => {
  2.   spyOn(window, 'updateResult') // and.stub() can be omitted because it is the default behaviour
  3.  
  4.   calculate('3+a')
  5.  
  6.   /**
  7.    * checks that our spy has been called atleast once in the calculate method call
  8.   */
  9.   expect(window.updateResult).toHaveBeenCalled()
  10.  
  11.   /**
  12.   * checks how many times our spy has been invoked
  13.   */
  14.   expect(window.updateResult).toHaveBeenCalledTimes(1)
  15. })
Spying on Prototypes

In jasmine spying on prototypes is also possible. We do this using the spyOn method where we can pass the class and the name of the prototype as arguments. In the below example we install a spy on the Calculator class multiply prototype and add expectations with the matchers discussed above. We can also use negators(.not.) with the spy matchers. 

  1. it('calls divide', () => {
  2.   /**
  3.    * spyOn method returns a spy instance which we can pass directly to our 
  4.    * expectation where we will use desired matchers
  5.   */
  6.   const spy = spyOn(Calculator.prototype, 'divide')
  7.  
  8.   calculate('8/4')
  9.  
  10.   // here we are checking how many times the 'divide' prototype of the calculator class is invoked
  11.   expect(spy).toHaveBeenCalledTimes(1)
  12.  
  13.   // here we check that the second number in the expression is passed to the spy instance
  14.   expect(spy).toHaveBeenCalledWith(4)
  15.  
  16.   // here we use the not negator to check if the expected value is not passed to the invoked spy
  17.   expect(spy).not.toHaveBeenCalledWith(8)
  18. })
The callThrough spy strategy

There are cases where we need to run an actual function for us to get or test whether a certain value is passed to our spy. We can use the callThrough strategy to tell a spy to send a call to the real implementation when invoked. In the below example we use callThrough on the multiply prototype spy so as to carry out an actual multiplication and check whether the actual product is passed to the updateResult spy. 

  1. it('calls updateResult (example using .and.callThrough())', () => {
  2.   const spy = spyOn(window, 'updateResult')
  3.  
  4.   /**
  5.   * Here, we install a spy on the multiply prototype and use
  6.   * callThrough to invoke the real implementation of the prototype
  7.   * 
  8.   * This will allow the real multiplication to be run that way we can
  9.   * test whether the final result was passed to the last updateResult
  10.   * function call
  11.   */
  12.   spyOn(Calculator.prototype, 'multiply').and.callThrough()
  13.  
  14.   calculate('5*5')
  15.  
  16.   expect(spy).toHaveBeenCalled()
  17.   expect(spy).toHaveBeenCalledWith(25)
  18. })
The callFake spy strategy

In rare cases, you may want to implement a custom callback as a fake installed on the spy. This 'fake' callback is called whenever the spy is invoked. 

  1. it('calls updateResult (example using .and.callFake())', () => {
  2.   const spy = spyOn(window, 'updateResult')
  3.  
  4.   /**
  5.   * Here, we install a spy and use callFake to call a specific custom 
  6.   * implementation when the  spy is invoked
  7.   * 
  8.   * This will be used in very rare cases and in ways looks like we are 
  9.   * testing the spy implementation itself
  10.   */
  11.   spyOn(Calculator.prototype, 'multiply').and.callFake((number) => {
  12. 	// here, the number argument is the second in the expression
  13. 	return number * 2
  14.   })
  15.  
  16.   calculate('5*4')
  17.  
  18.   expect(spy).toHaveBeenCalled()
  19.   // tests that 4 * 2 is 8
  20.   expect(spy).toHaveBeenCalledWith(8)
  21. })
The returnValue spy strategy

This strategy works the same way as callFake only it returns an actual defined value and not a callback. 

  1. it('calls updateResult (example using .and.returnValue())', () => {
  2.   const spy = spyOn(window, 'updateResult')
  3.  
  4.   /**
  5.   * Here, we install a spy and use returnValue to pass a final value that the spy returns
  6.   * 
  7.   * This strategy works the same way as the callFake, only it returns a value
  8.   */
  9.   spyOn(Calculator.prototype, 'multiply').and.returnValue('whatever [multiply] returns')
  10.  
  11.   calculate('5*4')
  12.  
  13.   expect(spy).toHaveBeenCalled()
  14.   expect(spy).toHaveBeenCalledWith('whatever [multiply] returns')
  15. })
The returnValues spy strategy

This strategy works the same way as returnValue only it accepts multiple values which are invoked in the order in which they are passed as arguments. 

  1. it('calls updateResult (example using .and.returnValues())', () => {
  2.   const spy = spyOn(window, 'updateResult')
  3.  
  4.   /**
  5.   * Here, returnValues is invoked in the order of the arguments passed
  6.   */
  7.   spyOn(Calculator.prototype, 'add').and.returnValues(5, 'whatever [add] returns')
  8.  
  9.   calculate('5+4')
  10.  
  11.   expect(spy).toHaveBeenCalled()
  12.   expect(spy).toHaveBeenCalledWith('whatever [add] returns')
  13. })
The throwError spy strategy

This strategy tells the spy installed to throw an error when invoked. This can be used to test whether error handling is working correctly in the method we are unit testing. 

  1. it('does not handle errors', () => {
  2.   /**
  3.    * This simply tells the spy to throw an error when invoked.
  4.   */
  5.   spyOn(Calculator.prototype, 'multiply').and.throwError('some error')
  6.  
  7.   /**
  8.    * Since the spy is throwing an error we can use the toThrowError matcher
  9.    * to check the exception thrown matches
  10.   */
  11.   expect(() => calculate('5*10')).toThrowError('some error')
  12. })
Using getters

In JavaScript, getters are used to bing and object property to a function whichc will be called when that property is accessed. Consider the example below where we want the latest added product. 

  1. const products = {
  2.     catalogue: [
  3.         { name: 'Product a', category: 'example 1', date_added: '2021-03-17' }, 
  4.         { name: 'Product c', category: 'example 3', date_added: '2021-03-19' },
  5.         { name: 'Product b', category: 'example 1', date_added: '2021-03-18' },
  6.     ],
  7.     get latest() {
  8.         if (this.catalogue.length === 0) {
  9.             return undefined
  10.         }
  11.  
  12.         // returns item with the latest date
  13.         return this.catalogue.reduce((possible_latest_item, current_item) => {
  14.             if (new Date(possible_latest_item.date_added) > new Date(current_item.date_added)) {
  15.                 return possible_latest_item
  16.             } else {
  17.                 return current_item
  18.             }
  19.         })
  20.     }
  21. };
  22.  
  23. console.log(products.latest);

In Jasmine, we cannot spy on getters and setters using spyOn. This is where we use spyOnProperty which can be used to install spyies on getters and setters. 

  1. describe('showVersion()', () => {
  2.   it('calls calculator.version getter', () => {
  3. 	/**
  4. 	 * Here we can use install a spy on the DOM instead of using a setup and teardown approach
  5. 	*/
  6. 	spyOn(document, 'getElementById').and.returnValue({
  7. 	  id: 'calculator-version',
  8. 	  innerText: null,
  9. 	})
  10.  
  11. 	/**
  12. 	 * This installs a spy on the getter 'version' since spyOn cannot work with getters.
  13. 	 * This method is similar ro spyOn, only it has a third argument which is the access type.
  14. 	 * This access type can be either 'get' or 'set', default value is 'get'
  15. 	 * 
  16. 	 * It can also be used on setters as well
  17. 	*/
  18. 	const spy_on_getter = spyOnProperty(Calculator.prototype, 'version', 'get')
  19.  
  20. 	showVersion()
  21.  
  22. 	expect(spy_on_getter).toHaveBeenCalled()
  23. 	expect(spy_on_getter).toHaveBeenCalledTimes(1)
  24.   })
  25. })
Calculator UI

Above, we added a spec file(main.spec.js) for the script(main.js) that handles the event listeners of the calculator UI.

Try it below:

Result: 3 Calculator Version 
  1. function calculate(input_value) {
  2.     /**
  3.      * Below regural expression(regex) is for matching our operators (+, -, * and /).
  4.      * In JavaScript we wrap regexes with a slash in the beginning and end of the expression.
  5.      * This regex was generated here: https://regex101.com/
  6.     */
  7.     const expression = /\+|\-|\*|\//;
  8.  
  9.     // Here we split the string from the input into an array
  10.     const numbers = input_value.split(expression);
  11.  
  12.     /**
  13.      * Get the first and second number in the arithmetic operation.
  14.      * Because they are strings we use parseInt() to convert them to numbers.
  15.     */
  16.     const first = parseInt(numbers[0]);
  17.     const second = parseInt(numbers[1]);
  18.  
  19.     /**
  20.      * The match helper is a string prototype that retrieves the result of matching a string against a
  21.      * regex. The result of it is returned as an array. In this case it will match the operator and return it.
  22.      * 
  23.      * More info: https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/String/match
  24.     */
  25.     const operator_match = input_value.match(expression);
  26.     let operator;
  27.  
  28.     if(Number.isNaN(first) || Number.isNaN(second) || operator_match === null) {
  29.         return updateResult('Operation not recognized');
  30.     } else {
  31.         // get the operator from the match result array
  32.         operator = operator_match[0];
  33.     }
  34.  
  35.     const calculator = new Calculator();
  36.     calculator.add(first);
  37.  
  38.     let result;
  39.  
  40.     switch (operator) {
  41.         case '+':
  42.             result = calculator.add(second);
  43.             break;
  44.         case '-':
  45.             result = calculator.subtract(second);
  46.             break;
  47.         case '*':
  48.             result = calculator.multiply(second);
  49.             break;
  50.         case '/':
  51.             result = calculator.divide(second);
  52.             break;
  53.         default:
  54.             break;
  55.     }
  56.  
  57.     return updateResult(result);
  58. };
  59.  
  60. function updateResult(calculator_result) {
  61.     const calculator_result_element = document.getElementById('calculator-result-span');
  62.  
  63.     if (calculator_result_element) {
  64.         calculator_result_element.innerText = `Result: ${calculator_result}`;
  65.     }
  66. };
  67.  
  68. function showVersion() {
  69.     const calculator =  new Calculator()
  70.  
  71.     const element = document.getElementById('calculator-version')
  72.  
  73.     // version is a getter in the Calculator class
  74.     element.innerText = calculator.version
  75. }
  76.  
  77. showVersion()
  78.  
  79. /**
  80.  * How to check an element exists in the DOM:
  81.  * 
  82.  * https://stackoverflow.com/questions/5629684/how-can-i-check-if-an-element-exists-in-the-visible-dom#answer-5629761
  83. */
  84. if (typeof(document.getElementById('calculator-input')) !== 'undefined' && document.getElementById('calculator-input') !== null) {
  85.     document.getElementById('calculator-input').addEventListener('keyup', (event) => {
  86.         if (event.key === 'Enter') {
  87.             return calculate(event.target.value);
  88.         }
  89.     })
  90. }

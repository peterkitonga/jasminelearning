Test Reports & Continuous Integration

In this section I learnt how to use the Karma runner and connecting unit tests to a Continuous Integration(CI) system. Here we used the node package manager npm to install all our dependencies. Below is the command used for the setup. 

  1. // initialize package.json
  2. npm init
  3.  
  4. // install dependencies
  5. npm install jasmine-core karma karma-jasmine karma-jasmine-matchers --save-dev
  6.  
  7. // install dependencies for headless browser(optional, used when using a CI system)
  8. npm install --save-dev puppeteer karma-chrome-launcher
The Karma Runner

To use the karma runner, we first need to define configurations in a karma.conf.js file. With this file we are able to configure how the karma runner will run our tests. Below is the configuration used. 

  1. module.exports = (config) => {
  2.     config.set({
  3.         color: true,
  4.         singleRun: true, // for CI system to run once and move on
  5.         frameworks: ['jasmine', 'jasmine-matchers'],
  6.         files: [ // order of files matters, same order is used to load them
  7.             'tests/custom/custom-matchers.js',
  8.             'assets/scripts/*.js',
  9.             'tests/specs/*.spec.js'
  10.         ],
  11.         plugins: ['karma-jasmine', 'karma-jasmine-matchers'],
  12.         reporters: ['dots'],
  13.     })
  14. };

With the configuration file in place, we can now setup a script in the package.json file like below. Once the script is in place we can run npm test which will start our karma runner. 

  1. "scripts": {
  2.     "test": "karma start"
  3. }
Coverage Reports

A coverage report tells us which functions and lines of code are being executed when running our tests. This tells us whether our tests are testing exactly what we intend. For this we install a dependency for coverage reports called istanbul npm install istanbul --save-dev. Note: This dependency is deprecated, use the new package npm install nyc --save-dev

We also need the karma CLI which we install via the command npm install -g karma-cli, this will install it globaly. We also install the karma coverage plugin using the command npm install karma-coverage --save-dev. Once those are in place, we add the karma-coverage plugin to the plugins array and configure the coverageReporter and preprocessors keys like below. Thereafter we run npm test command which will run the tests and generate a coverage report under the tests/coverage/ folder. Here we can open the index.html file under the html folder generated. 

  1. module.exports = (config) => {
  2.   config.set({
  3. 	color: true,
  4. 	singleRun: true,
  5. 	frameworks: ['jasmine', 'jasmine-matchers'],
  6. 	preprocessors: {
  7. 	  // tells the coverage report where to report on
  8. 	  'assets/scripts/*.js': ['coverage'],
  9. 	  'tests/specs/*.js': ['coverage']
  10. 	},
  11. 	files: [
  12. 	  'tests/helpers/custom-matchers.js',
  13. 	  'assets/scripts/*.js',
  14. 	  'tests/specs/*.spec.js'
  15. 	],
  16. 	plugins: [
  17. 	  'karma-jasmine', 
  18. 	  'karma-jasmine-matchers', 
  19. 	  'karma-chrome-launcher',
  20. 	  'karma-coverage'
  21. 	],
  22. 	reporters: ['dots', 'coverage'], // 'coverage' value is added to as to enable the coverage reporter
  23. 	browsers: ['ChromeHeadless'],
  24. 	coverageReporter: {
  25. 	  // tells the coverage reporter where and how to generate the reports
  26. 	  dir: 'tests/coverage/',
  27. 	  reporters: [
  28. 		{ type: 'html', subdir: 'html' }
  29. 	  ]
  30. 	}
  31.   })
  32. };

Connecting to Travis CI and Configuring the build

Here, we configure the travis manifest file .travis.yml. This is where we pass the language in use for the test runner, addons, scripts and setup instructions for Travis CI to run inorder to execute our tests. The below file is an example of a config. 

  1. language: node_js
  2. node_js:
  3.   - "14.15.1"
  4. install:
  5.   - npm install
  6. script:
  7.   - npm test
  8. addons:
  9.   chrome: stable
  10. before_install:
  11.   - google-chrome-stable --headless --disable-gpu --remote-debugging-port=9222 http://localhost &
  12. before_script:
  13.   - "sudo chown root /opt/google/chrome/chrome-sandbox"
  14.   - "sudo chmod 4755 /opt/google/chrome/chrome-sandbox"

With this in place we first need to create an account or sign in to Travis CI and enable our project for travis to run the build. Once the build passes we can copy and add the travis build status badge to our readme.md file. This badge will always let us know whether our tests are passing everytime a commit is pushed to the repository.

Calculator UI

Above, we added a spec file(main.spec.js) for the script(main.js) that handles the event listeners of the calculator UI.

Try it below:

Result: 3 Calculator Version 
  1. function calculate(input_value) {
  2.     /**
  3.      * Below regural expression(regex) is for matching our operators (+, -, * and /).
  4.      * In JavaScript we wrap regexes with a slash in the beginning and end of the expression.
  5.      * This regex was generated here: https://regex101.com/
  6.     */
  7.     const expression = /\+|\-|\*|\//;
  8.  
  9.     // Here we split the string from the input into an array
  10.     const numbers = input_value.split(expression);
  11.  
  12.     /**
  13.      * Get the first and second number in the arithmetic operation.
  14.      * Because they are strings we use parseInt() to convert them to numbers.
  15.     */
  16.     const first = parseInt(numbers[0]);
  17.     const second = parseInt(numbers[1]);
  18.  
  19.     /**
  20.      * The match helper is a string prototype that retrieves the result of matching a string against a
  21.      * regex. The result of it is returned as an array. In this case it will match the operator and return it.
  22.      * 
  23.      * More info: https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/String/match
  24.     */
  25.     const operator_match = input_value.match(expression);
  26.     let operator;
  27.  
  28.     if(Number.isNaN(first) || Number.isNaN(second) || operator_match === null) {
  29.         return updateResult('Operation not recognized');
  30.     } else {
  31.         // get the operator from the match result array
  32.         operator = operator_match[0];
  33.     }
  34.  
  35.     const calculator = new Calculator();
  36.     calculator.add(first);
  37.  
  38.     let result;
  39.  
  40.     switch (operator) {
  41.         case '+':
  42.             result = calculator.add(second);
  43.             break;
  44.         case '-':
  45.             result = calculator.subtract(second);
  46.             break;
  47.         case '*':
  48.             result = calculator.multiply(second);
  49.             break;
  50.         case '/':
  51.             result = calculator.divide(second);
  52.             break;
  53.         default:
  54.             break;
  55.     }
  56.  
  57.     return updateResult(result);
  58. };
  59.  
  60. function updateResult(calculator_result) {
  61.     const calculator_result_element = document.getElementById('calculator-result-span');
  62.  
  63.     if (calculator_result_element) {
  64.         calculator_result_element.innerText = `Result: ${calculator_result}`;
  65.     }
  66. };
  67.  
  68. function showVersion() {
  69.     const calculator =  new Calculator()
  70.  
  71.     const element = document.getElementById('calculator-version')
  72.  
  73.     // version is a getter in the Calculator class
  74.     calculator.version.then(version => {
  75.         element.innerText = version
  76.     })
  77. }
  78.  
  79. /**
  80.  * How to check an element exists in the DOM:
  81.  * 
  82.  * https://stackoverflow.com/questions/5629684/how-can-i-check-if-an-element-exists-in-the-visible-dom#answer-5629761
  83. */
  84. if (typeof(document.getElementById('calculator-input')) !== 'undefined' && document.getElementById('calculator-input') !== null) {
  85.     document.getElementById('calculator-input').addEventListener('keyup', (event) => {
  86.         if (event.key === 'Enter') {
  87.             return calculate(event.target.value);
  88.         }
  89.     })
  90. }
